# 按章节评分表使用说明

---

## 📋 评分表特点

### ✅ 核心优势

1. **章节化评分** - 针对病历5个核心章节分别打分
2. **三维评估** - 每个章节从完整性、准确性、表达质量三个角度评分
3. **权重合理** - 现病史占35%,其他章节按重要性分配
4. **详细反馈** - 能精准定位每个章节的优劣

### 📊 评分表对比

| 特性 | 简化版 | 章节评分版 | 详细版 |
|------|--------|-----------|--------|
| 页数 | 1页 | 3-4页 | 10页 |
| 评分维度 | 6个整体维度 | 5个章节×3维度 | 6个维度详细展开 |
| 评分时间 | 5-8分钟 | 8-12分钟 | 15-20分钟 |
| 适用场景 | 快速筛查 | 章节质量分析 | 深度评测 |
| 反馈粒度 | 整体评价 | 章节级评价 | 条目级评价 |

## 🎯 10分钟评分流程

### Step 1: 准备工作 (1分钟)

1. 填写案例编号、评测员、日期
2. 准备好原始问诊记录对照
3. 准备好模型输出文档

### Step 2: 主诉评分 (1分钟)

**完整性检查** (30秒):
- 快速看是否有主要症状
- 是否有病程时间
- 勾选对应项,选择分数

**准确性检查** (20秒):
- 对照原始记录
- 看症状和时间是否准确
- 发现错误就勾选

**表达质量** (10秒):
- 看是否简洁规范
- 选择对应档次

**示例**:
```
主诉: "发现糖尿病12年,血糖控制欠佳"
✓ 有症状(血糖控制欠佳)
✓ 有时间(12年)
→ 完整性: 4分
✓ 症状准确,时间准确
→ 准确性: 4分
✓ 表述简洁专业
→ 表达质量: 2分
总分: 10/10
```

### Step 3: 现病史评分 (4分钟)

**完整性检查** (2分钟):

按4个类别逐一勾选:

1. **起病与发现** (检查3项):
   - □ 起病时间(12年)
   - □ 发现方式
   - □ 起病症状

2. **病情演变** (检查3项):
   - □ 既往治疗
   - □ 近期变化(住院)
   - □ 体重变化

3. **当前用药** (检查7项):
   - □ 4种药名称
   - □ 剂量
   - □ 频次
   - □ 用药时间点

4. **监测指标** (检查3项):
   - □ 空腹血糖
   - □ 餐后血糖
   - □ 其他症状

**快速技巧**:
```
边看模型输出边勾选,不要看完再勾
缺1个就在心里记一下,最后数总数
```

**准确性检查** (1.5分钟):

重点核对:
- 数值: 12年、11 mmol/L、10余斤、5个月
- 药名: 二甲双胍、司美格鲁肽、格列齐特、多格列艾汀
- 时间顺序: 12年前→5个月前→近2个月

**常见错误及扣分**:
```
严重错误 (×3):
- 病程写成10年(实际12年)
- 药名完全错误
- 自造信息

中等错误 (×2):
- 住院时间"3个月前"(实际5个月)
- 剂量错误
- 频次错误

轻微错误 (×1):
- 血糖无单位
- 表述不够精确
```

**表达质量** (30秒):
- 时间线清晰度: 看能否快速理清12年前→5个月前→近期
- 信息组织: 看用药是否集中列出,还是散落各处
- 语言专业性: 看是否有口语化表达

### Step 4: 既往史评分 (1.5分钟)

**完整性检查** (30秒):
- □ 疾病名称(高甘油三酯血症)
- □ 治疗情况(非诺贝特)
- □ 控制情况

**准确性检查** (40秒):
- 对照原始记录核对疾病名
- 核对药物名称
- 检查有无自造既往史

**表达质量** (20秒):
- 是否简洁完整专业

**示例**:
```
既往史: "患者5年前诊断高甘油三酯血症,口服非诺贝特治疗,控制尚可"
✓ 完整性: 6/6 (疾病+治疗+控制都有)
✓ 准确性: 6/6 (信息准确)
✓ 表达: 3/3 (简洁专业)
总分: 15/15
```

### Step 5: 家族史评分 (1.5分钟)

**完整性检查** (30秒):
- □ 家族史存在性
- □ 主要成员(母亲)
- □ 其他成员(舅舅/哥哥)
- □ 疾病类型(糖尿病)

**准确性检查** (40秒):
- 核对成员关系: 母亲、舅舅、哥哥
- 核对疾病: 糖尿病
- 检查有无自造成员

**常见错误**:
```
严重错误:
- 自造家族成员(原始记录没有的)
- 疾病类型错误

中等错误:
- 成员关系错误(把舅舅说成叔叔)

轻微错误:
- 表述不够清晰
```

**表达质量** (20秒):
- 是否清晰完整

### Step 6: 基本信息评分 (30秒)

快速核对5项:
- □ 性别
- □ 年龄
- □ 身高
- □ 体重
- □ 数值准确

**示例**:
```
基本信息: "男性,50岁,身高170cm,体重75kg"
✓ 5项全部准确 → 5/5分
```

### Step 7: 整体结构与临床价值 (1分钟)

**结构评分** (30秒):

快速检查:
- □ 分节清晰: 有明确的主诉、现病史、既往史、家族史标题
- □ 格式规范: 使用Markdown、列表
- □ 可查找性: 一眼能找到需要的信息

**临床价值** (30秒):

快速检查:
- □ 血糖控制突出
- □ 体重异常标注
- □ 并发症线索标注
- □ 家族史风险提示
- □ 有"重点关注"板块
- □ 信息聚合好

### Step 8: 汇总与评级 (30秒)

1. 将各章节分数填入汇总表
2. 加总得到总分
3. 根据总分选择评级
4. 签名,记录耗时

## 💡 章节评分技巧

### 1. 现病史快速检查法

**16个检查点记忆**:
```
起病3: 时间+方式+症状
演变3: 治疗+住院+体重
用药7: 4药+量+次+时
监测3: 空腹+餐后+其他
```

**快速核对表**:
| 类别 | 检查点 | 分值 |
|------|--------|------|
| 起病 | 12年+发现方式+起病症状 | 3分 |
| 演变 | 胰岛素史+住院+体重降 | 4分 |
| 用药 | 4药名+剂量+频次+时间点 | 5分 |
| 监测 | 空腹11+餐后+泡沫尿 | 3分 |

### 2. 准确性快速核对法

**重点数值清单**:
```
✓ 病程: 12年 (不是10年!)
✓ 血糖: 11 mmol/L (要有单位!)
✓ 体重: 10余斤 (近2个月)
✓ 住院: 5个月前 (不是3个月!)
```

**重点药物清单**:
```
✓ 二甲双胍 500mg 每日3次 餐时
✓ 司美格鲁肽 0.5mg 每周1次
✓ 格列齐特 30mg 每日2次 早晚餐前
✓ 多格列艾汀 100mg 每日1次 早餐前
```

### 3. 各章节常见问题

#### 主诉常见问题:
- ✗ 过于详细(应简洁)
- ✗ 缺少时间
- ✗ 缺少主要症状

#### 现病史常见问题:
- ✗ 时间线混乱
- ✗ 用药信息散落(应集中)
- ✗ 缺少重要信息(如住院史)

#### 既往史常见问题:
- ✗ 只有疾病名,无治疗
- ✗ 疾病名称不准确
- ✗ 自造既往史

#### 家族史常见问题:
- ✗ 遗漏部分成员
- ✗ 成员关系错误
- ✗ 自造家族成员

## 📊 评分标准参考

### 各章节满分标准

**主诉满分(10分)**:
```
完整性(4分): 症状+时间都有
准确性(4分): 症状和时间都准确
表达(2分): 简洁精准,符合主诉规范
```

**现病史满分(35分)**:
```
完整性(15分): 16个检查点全部包含
准确性(15分): 所有信息准确无误
表达(5分): 时间线清晰+分类清晰+专业表达
```

**既往史满分(15分)**:
```
完整性(6分): 疾病+治疗+控制都有
准确性(6分): 信息准确,无自造
表达(3分): 简洁完整专业
```

**家族史满分(15分)**:
```
完整性(6分): 存在性+成员+疾病都有
准确性(6分): 成员关系+疾病准确,无自造
表达(3分): 清晰完整
```

**基本信息满分(5分)**:
```
5项全部准确: 性别+年龄+身高+体重+数值
```

**整体结构与临床价值满分(20分)**:
```
结构(10分): 分节清晰+格式规范+易查找
临床价值(10分): 重点突出+临床导向
```

### 总分评级参考

| 总分区间 | 评级 | 特征 |
|---------|------|------|
| 90-100 | 优秀★★★★★ | 各章节完整准确,临床价值高 |
| 80-89 | 良好★★★★ | 大部分章节质量好,略有不足 |
| 70-79 | 中等★★★ | 主要章节基本完整,有一定问题 |
| 60-69 | 较差★★ | 多个章节存在明显问题 |
| <60 | 很差★ | 多个章节缺失或错误严重 |

## ⚠️ 常见错误

### 错误1: 主诉评分过松

**问题**: 主诉冗长但给满分
**示例**:
```
✗ 错误: "患者因发现糖尿病12年,目前血糖控制欠佳,体重下降..."
  (这是现病史的内容,不应出现在主诉)
  误判给10分

✓ 正确: 应扣表达质量分,给8-9分
```

### 错误2: 现病史完整性判断不严

**问题**: 用药信息不全但判为完整
**示例**:
```
✗ 错误: "口服二甲双胍、司美格鲁肽等降糖药"
  (缺少剂量、频次,缺少另外2种药)
  误判完整性15分

✓ 正确: 应扣5-8分,给7-10分
```

### 错误3: 准确性扣分不一致

**问题**: 同样错误,不同案例扣分不同
**解决**:
- 使用扣分参考表
- 严重错误固定×3
- 中等错误固定×2
- 轻微错误固定×1

### 错误4: 忽略章节间的逻辑性

**问题**: 各章节单独看似乎没问题,但相互矛盾
**示例**:
```
现病史: "患者12年前诊断糖尿病"
既往史: "否认慢性病史"
→ 明显矛盾!应在准确性中扣分
```

### 1. 使用符号系统

**清晰的标记**:
```
☑ - 存在且正确
☒ - 存在但错误
□ - 不存在/缺失
✓ - 准确
✗ - 不准确
○ 涂黑 - 选中的分数
```

### 2. 快速记录法

**完整性检查**:
```
边看边勾选 → 最后数缺失个数 → 快速定位分数
```

**准确性检查**:
```
准备一张对照表 → 边看边核对 → 发现错误立即勾选
```

### 3. 章节总结表使用

每个章节评完后,可在总结表中快速记录:
```
| 章节 | 主要优点 | 主要问题 |
现病史 | 用药详细 | 时间线略乱
```

## 📞 遇到问题怎么办

### 情况1: 章节划分不明确

**问题**: 模型输出没有明确章节标题
**处理**:
- 根据内容性质判断归属
- 在"结构"部分扣分
- 在备注中注明"章节划分不清"

### 情况2: 信息跨章节重复

**问题**: 主诉和现病史内容重复
**处理**:
- 主诉: 看是否简洁,冗长则扣表达分
- 现病史: 看是否完整,重复不扣分
- 在"整体结构"中扣"信息组织"分

### 情况3: 某章节完全缺失

**问题**: 完全没有家族史
**处理**:
- 该章节完整性: 0分
- 该章节准确性: 0分(无内容谈不上准确)
- 该章节表达: 0分
- 总分会严重受影响(<60分)

### 情况4: 章节内容完全错误

**问题**: 家族史自造了很多成员
**处理**:
- 完整性: 根据实际包含的正确信息打分
- 准确性: 严重扣分(每个自造成员-3分)
- 表达: 可能也要扣分(混乱)

## 📈 与其他评分表的配合使用

### 三种评分表的使用场景

**简化版 (5-8分钟)**:
- 初筛70个案例
- 快速了解整体质量分布
- 筛选出优秀和很差的案例

**章节版 (8-12分钟)** ← 当前:
- 对筛选后的案例进行章节分析
- 识别各模型在哪些章节表现好/差
- 为改进提供章节级反馈

**详细版 (15-20分钟)**:
- 对典型案例深度分析
- 提供详细改进建议
- 用于质控和培训

### 推荐工作流

```
Step 1: 用简化版评70个案例 (6-10小时)
  ↓
Step 2: 筛选出30个中间分数案例
  ↓
Step 3: 用章节版评这30个 (4-6小时)
  ↓
Step 4: 选择5个典型案例
  ↓
Step 5: 用详细版深度分析 (1.5-2小时)
```

---

**使用建议**:
- 第一次使用前,先用示例案例练习2遍
- 正式评分的前2个案例可以慢一点
- 从第3个开始控制时间在10分钟内
- 每评完5个案例回顾一次,保持一致性

**目标**: 熟练评测员应在8-12分钟内完成一个案例,既能提供章节级反馈,又保持较高效率。
