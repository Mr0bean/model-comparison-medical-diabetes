# 病历评测Prompt模板

## 文件结构

```
Prompts/
├── README.md                # 本文件
├── 01-准确性.md             # 准确性评测 (40分)
├── 02-逻辑性.md             # 逻辑性评测 (25分)
├── 03-完整性.md             # 完整性评测 (15分)
├── 04-格式规范性.md         # 格式规范性评测 (15分)
└── 05-语言表达.md           # 语言表达评测 (5分)
```

## 评分体系

| 维度 | 分值 | 文件 |
|------|------|------|
| 准确性 | 40分 | 01-准确性.md |
| 逻辑性 | 25分 | 02-逻辑性.md |
| 完整性 | 15分 | 03-完整性.md |
| 格式规范性 | 15分 | 04-格式规范性.md |
| 语言表达 | 5分 | 05-语言表达.md |
| **总分** | **100分** | |

## 使用方法

### 1. 单维度评测

每个prompt文件独立调用，输入：
- 原始对话
- 生成的医疗报告

输出：
- 该维度得分
- 具体问题描述

### 2. 聚合成完整报告

依次调用5个prompt文件，收集各维度评测结果后，聚合为完整评测报告：

```json
{
  "model_name": "模型名称",
  "total_score": "总分 (0-100)",
  "dimensions": {
    "accuracy": {
      "score": "得分 (满分40)",
      "issues": "具体问题"
    },
    "logic": {
      "score": "得分 (满分25)",
      "issues": "具体问题"
    },
    "completeness": {
      "score": "得分 (满分15)",
      "issues": "具体问题"
    },
    "formatting": {
      "score": "得分 (满分15)",
      "issues": "具体问题"
    },
    "language": {
      "score": "得分 (满分5)",
      "issues": "具体问题"
    }
  },
  "critical_feedback": "综合改进建议"
}
```

**聚合步骤：**
1. 依次调用01-05.md，传入相同的[原始对话]和[医疗报告]
2. 收集5个维度的评测结果
3. 计算总分 = 各维度得分之和
4. 汇总所有issues和feedback
5. 生成完整的JSON报告
