# 医疗AI模型输出评估标准（100分制）

## 📊 评分维度总览

| 维度 | 分值 | 权重 | 重点考察 |
|------|------|------|----------|
| **1. 信息准确性** | 30分 | 30% | 是否有错误、曲解 |
| **2. 信息完整性** | 25分 | 25% | 关键信息是否遗漏 |
| **3. 临床实用性** | 20分 | 20% | 对诊疗决策的帮助 |
| **4. 结构清晰度** | 15分 | 15% | 组织是否合理易读 |
| **5. 语言专业性** | 10分 | 10% | 表达是否专业简洁 |
| **总分** | **100分** | **100%** | - |

---

## 📋 详细评分标准

### 1️⃣ 信息准确性（30分）

**核心原则**：所有信息必须与原始对话一致，不能有任何错误或曲解。

#### 评分等级

| 分数 | 等级 | 标准描述 |
|------|------|----------|
| **28-30分** | 优秀 | ✓ 所有信息完全准确<br>✓ 数值、时间、药名、剂量完全正确<br>✓ 无任何错误 |
| **24-27分** | 良好 | ✓ 绝大部分准确<br>✗ 1-2处轻微不精确（如"50多岁"写成"约50岁"） |
| **18-23分** | 中等 | ✓ 大部分准确<br>✗ 3-4处小错误，或1处中等错误 |
| **12-17分** | 较差 | ✗ 多处错误<br>✗ 有关键信息错误 |
| **0-11分** | 很差 | ✗ 大量严重错误<br>✗ 关键信息严重失真 |

#### 错误扣分参考

| 错误类型 | 严重程度 | 扣分 | 示例 |
|----------|----------|------|------|
| 数值完全错误 | 🔴 严重 | -5分 | "血糖11" → "血糖7" |
| 药名错误 | 🔴 严重 | -5分 | "二甲双胍" → "格列美脲" |
| 无中生有 | 🔴 严重 | -5分 | 编造不存在的检查结果 |
| 时间点大错（>5年） | 🔴 严重 | -4分 | "12年" → "5年" |
| 药物剂量/频次错误 | 🟡 中等 | -3分 | "早晚各1次" → "每日1次" |
| 时间点小错（1-2年） | 🟡 中等 | -2分 | "12年" → "10年" |
| 数值不精确 | 🟢 轻微 | -1分 | "11 mmol/L" → "约11" |

#### ✅ 检查清单

- [ ] 患者基本信息（性别、年龄、身高、体重）
- [ ] 病程时间
- [ ] 所有药物名称、剂量、频次
- [ ] 所有检查数值及单位
- [ ] 时间节点（起病、住院、体重变化等）
- [ ] 家族史人员关系
- [ ] 症状描述

### 2️⃣ 信息完整性（25分）

**核心原则**：所有关键信息都应被提取，不能有重要遗漏。

#### 评分等级

| 分数 | 等级 | 标准描述 |
|------|------|----------|
| **23-25分** | 优秀 | ✓ 所有关键信息完整<br>✓ 次要信息也基本覆盖<br>✓ 无重要遗漏 |
| **19-22分** | 良好 | ✓ 关键信息完整<br>✗ 遗漏1-2个次要信息 |
| **14-18分** | 中等 | ✓ 大部分关键信息<br>✗ 遗漏3-4个信息或1个较重要信息 |
| **8-13分** | 较差 | ✗ 关键信息不完整<br>✗ 遗漏多个重要信息 |
| **0-7分** | 很差 | ✗ 大量关键信息缺失 |

#### 必需信息清单（重要性分级）

**⭐⭐⭐ 高优先级（必须包含）**
- [ ] 主诉/就诊原因
- [ ] 病程时间
- [ ] 当前用药方案（药名+剂量+频次）
- [ ] 近期检查结果（血糖等）
- [ ] 重要症状（如体重下降、泡沫尿等）

**⭐⭐ 中优先级（应该包含）**
- [ ] 既往治疗史（如曾用胰岛素）
- [ ] 并发症/合并症
- [ ] 家族史
- [ ] 近期病情变化（如住院史）

**⭐ 低优先级（最好包含）**
- [ ] 生活方式（运动、饮食）
- [ ] 依从性信息
- [ ] 次要症状

#### 评分计算

```
覆盖率 = 提取的信息点数 / 应有的信息点数

25分：覆盖率 ≥ 95%（遗漏≤1个次要信息）
22分：覆盖率 90-94%（遗漏1-2个次要信息）
18分：覆盖率 80-89%（遗漏3-4个信息）
14分：覆盖率 70-79%（遗漏较多或1个重要信息）
10分：覆盖率 60-69%
<10分：覆盖率 < 60%
```

### 3️⃣ 临床实用性（20分）

**核心原则**：信息组织应突出临床关键点，有助于快速做出诊疗决策。

#### 评分等级

| 分数 | 等级 | 标准描述 |
|------|------|----------|
| **18-20分** | 优秀 | ✓ 重点信息突出（警示信息、异常指标）<br>✓ 相关信息聚合（如所有用药集中）<br>✓ 有助于快速决策<br>✓ 关键风险点有强调 |
| **15-17分** | 良好 | ✓ 重要信息基本突出<br>✓ 信息聚合较好<br>✗ 个别关键点不够明显 |
| **11-14分** | 中等 | ✓ 信息平铺，重点不突出<br>✗ 需医生自行筛选<br>✗ 帮助有限 |
| **6-10分** | 较差 | ✗ 重要信息淹没在次要信息中<br>✗ 缺少临床思维 |
| **0-5分** | 很差 | ✗ 完全无临床导向<br>✗ 无法支持决策 |

#### 临床关键点检查（每突出1项 +2-3分）

**🔴 高度关注（必须突出）**
- [ ] 血糖控制情况及控制目标
- [ ] 体重异常变化（尤其是快速下降）
- [ ] 当前完整用药方案
- [ ] 并发症线索（如泡沫尿→肾病）

**🟡 重要关注（应该突出）**
- [ ] 家族史（强家族史影响预后）
- [ ] 既往住院史（反映病情严重度）
- [ ] 依从性问题
- [ ] 血糖控制不佳的可能原因

#### 优秀示例特征

```markdown
## ⚠️ 重点关注
1. **血糖控制不佳**: 空腹11 mmol/L (目标<7)，4联用药仍控制不良
2. **体重异常下降**: 2个月降10余斤，需排除酮症
3. **疑似肾病**: 持续泡沫尿12年，建议查尿蛋白
4. **强家族史**: 母系及同胞多人患病

## 当前用药方案（4联）
1. 二甲双胍 早晚各1次
2. 司美格鲁肽 每周1次
3. 格列齐特 早2粒
4. 多格列艾汀 早晚各1次
```

### 4️⃣ 结构清晰度（15分）

**核心原则**：信息组织清晰、层次分明、易于查找。

#### 评分等级

| 分数 | 等级 | 标准描述 |
|------|------|----------|
| **14-15分** | 优秀 | ✓ 分节清晰（基本信息、主诉、现病史等）<br>✓ 层次分明，使用恰当的标题/列表<br>✓ 时间线清晰<br>✓ 一目了然 |
| **11-13分** | 良好 | ✓ 分节基本清晰<br>✗ 个别部分略显混乱<br>✓ 整体可用性好 |
| **8-10分** | 中等 | ✓ 有分节但不够清晰<br>✗ 部分信息归类不当<br>✗ 需仔细阅读 |
| **4-7分** | 较差 | ✗ 分节混乱或缺失<br>✗ 信息堆砌，逻辑不清 |
| **0-3分** | 很差 | ✗ 完全无结构<br>✗ 杂乱无章 |

#### 结构评估要点

**✅ 优秀结构特征**
- [ ] 有清晰的章节划分（基本信息、主诉、现病史、既往史等）
- [ ] 使用标题层级（#, ##, ###）
- [ ] 相关信息归类合理（如所有用药在一起）
- [ ] 时间线清晰（起病→治疗→变化）
- [ ] 使用列表/表格组织复杂信息
- [ ] 重点信息有标注（加粗、emoji等）

**❌ 不良结构特征**
- [ ] 信息堆砌成大段文本
- [ ] 时间线混乱（事件顺序不清）
- [ ] 相关信息分散（用药信息散落各处）
- [ ] 无标题或标题不清晰
- [ ] 难以快速定位关键信息

### 5️⃣ 语言专业性（10分）

**核心原则**：语言专业、简洁、流畅，符合医疗文书规范。

#### 评分等级

| 分数 | 等级 | 标准描述 |
|------|------|----------|
| **9-10分** | 优秀 | ✓ 医学术语准确<br>✓ 表达简洁无冗余<br>✓ 语句通顺专业 |
| **7-8分** | 良好 | ✓ 表达基本专业<br>✗ 个别用词不够精确或略显冗余 |
| **5-6分** | 中等 | ✓ 表达基本清楚<br>✗ 有一定赘述或用词不当 |
| **3-4分** | 较差 | ✗ 表达不清晰<br>✗ 多处赘述或用词错误 |
| **0-2分** | 很差 | ✗ 表达混乱，语病多 |

#### 常见问题及扣分

**❌ 冗余表达（每处 -0.5分）**
- "患者的糖尿病疾病" → ✅ "糖尿病"
- "进行了住院治疗" → ✅ "住院治疗"
- "血糖数值为11" → ✅ "血糖11"

**❌ 口语化（每处 -1分）**
- "有点高" → ✅ "升高"
- "吃药" → ✅ "服药"
- "尿里有泡泡" → ✅ "泡沫尿"

**❌ 不专业（每处 -1分）**
- "血糖不好" → ✅ "血糖控制不佳"
- "打针" → ✅ "注射胰岛素"
- "糖尿病很严重" → ✅ "糖尿病控制不良"

**✅ 专业简洁示例**
- "确诊2型糖尿病12年"
- "近期空腹血糖11 mmol/L"
- "口服降糖药4种"
- "否认高血压、冠心病病史"

## 🎯 快速评分流程

### Step 1: 准备（1分钟）
1. 阅读原始问诊记录
2. 标记关键信息点（10-15个）
3. 记录关键数值（血糖、体重、病程等）

### Step 2: 逐维度评分（5-7分钟）

**① 准确性（30分）** - 错误扣分法
- 从30分开始
- 逐项核对，发现错误即扣分
- 参考"错误扣分表"

**② 完整性（25分）** - 覆盖率法
- 统计原始记录信息点总数
- 统计模型提取的信息点数
- 计算覆盖率并对照评分表

**③ 实用性（20分）** - 关键点法
- 检查"临床关键点清单"
- 每突出1项 +2-3分
- 信息组织有助决策 +额外分

**④ 结构（15分）** - 整体印象法
- 整体浏览，评估"好不好用"
- 检查"结构评估要点"
- 对照等级描述打分

**⑤ 语言（10分）** - 问题计数法
- 从10分开始
- 发现冗余/口语化/不专业即扣分
- 参考"常见问题及扣分"

### Step 3: 总分计算与复核（1分钟）
- 汇总5个维度得分
- 检查总分是否合理（与整体印象是否一致）
- 记录典型优点和问题

## 💡 评分注意事项

### ✅ DO - 应该这样做

1. **客观公正**: 基于标准评分，不受个人偏好影响
2. **前后一致**: 同样的错误给予同样的扣分
3. **仔细核对**: 逐项核对，不遗漏任何错误
4. **临床视角**: 站在临床医生角度评估实用性
5. **记录依据**: 记录扣分/加分理由，便于复核

### ❌ DON'T - 避免这样做

1. **不要凭感觉打分**: 必须对照标准
2. **不要过于宽松或严格**: 严格按照扣分标准
3. **不要受格式影响**: 只要信息对，格式可以有差异
4. **不要因疲劳降低标准**: 休息后再继续评分
5. **不要受模型品牌影响**: 盲评，不看模型名称

## 📚 附录：评分示例

### 示例1: 优秀输出（92分）

**原始问诊**：
```
患者男，50多岁，182cm，64kg。12年前小便泡沫多发现糖尿病，
用过胰岛素后停，现用二甲双胍早晚各1次、司美格鲁肽每周1次、
格列齐特早2粒、多格列艾汀早晚各1次。空腹血糖11，未测餐后。
5个月前住院，出院后2月内瘦了10多斤。母亲、舅舅、哥哥都有糖尿病。
甘油三酯高，未治疗。
```

**模型输出**（得分92分）：
```markdown
# 患者基本信息
- 性别：男
- 年龄：50-55岁
- 身高：182 cm
- 体重：64 kg
- BMI：19.3 kg/m² (偏低)

## ⚠️ 重点关注
1. **血糖控制不佳**：空腹血糖11 mmol/L（目标<7），4联用药仍控制不良
2. **体重异常下降**：2个月内下降10余斤，需警惕并发症或血糖控制差
3. **疑似肾脏并发症**：小便泡沫持续12年，强烈建议检查尿蛋白
4. **强家族史**：母系及同胞多人患病，疾病进展风险高

## 主诉
配药

## 现病史

### 起病
- **病程**：12年前
- **首发症状**：小便泡沫多
- **诊断**：血糖升高，诊断为2型糖尿病

### 治疗经过
**既往治疗**：
- 曾使用胰岛素（已停用）

**当前用药方案（4联）**：
1. 二甲双胍 - 早晚各1次（剂量不详）
2. 司美格鲁肽（GLP-1RA）- 每周1次
3. 格列齐特（磺脲类）- 早2粒
4. 多格列艾汀（DPP-4抑制剂）- 早晚各1次

### 近期病情变化
- **5个月前**：因血糖控制不佳住院治疗
- **出院后2月内**：体重下降10余斤
- **目前**：体重相对稳定

### 血糖监测
- 空腹血糖：11 mmol/L（控制不佳）
- 餐后血糖：未监测

## 既往史
- **高脂血症**：甘油三酯升高，未服药治疗

## 家族史
- 母亲：糖尿病
- 舅舅：糖尿病
- 哥哥：糖尿病
（强家族史，3位一级或二级亲属患病）

## 生活方式
- 运动：未提及
- 饮食：未提及
- 烟酒：未提及
```

**评分详情**：
- ✅ 准确性 30/30：所有信息准确，无错误
- ✅ 完整性 24/25：覆盖率98%，仅缺少烟酒史
- ✅ 实用性 20/20：重点突出，临床导向强，有风险提示
- ✅ 结构 15/15：结构清晰，层次分明，易读性强
- ✅ 语言 13/10：表达专业，简洁流畅（满分只给10分）

**总分：92/100** (实际按100分上限)

### 示例2: 中等输出（68分）

**模型输出**（得分68分）：
```markdown
患者基本信息：男性，约50岁，身高182cm，体重64kg。

主诉：来院配药。

现病史：患者于12年前因小便有泡沫就诊，检查发现血糖升高，
诊断为糖尿病。曾经使用过胰岛素治疗，目前使用二甲双胍、
司美格鲁肽、格列齐特和多格列艾汀进行治疗。最近检查空腹血糖为11。
患者在5个月前曾经住院，出院后体重下降。

既往史：有甘油三酯增高的情况。

家族史：母亲、舅舅和哥哥都患有糖尿病。
```

**评分详情**：
- ⚠️ 准确性 24/30：扣6分
  - "约50岁"不精确(-1)，应为"50-55岁"
  - 缺少血糖单位(-1)
  - 缺少用药剂量和频次(-4)

- ⚠️ 完整性 17/25：扣8分，覆盖率约75%
  - 遗漏：用药剂量频次、餐后未测、体重下降具体数值、病程时间点

- ⚠️ 实用性 11/20：扣9分
  - 重点不突出，信息平铺
  - 未强调血糖控制不佳
  - 未提及并发症风险
  - 未聚合关键信息

- ✅ 结构 10/15：扣5分
  - 有分节但过于简单
