# 简化评分表使用说明

## 📋 评分表特点

### ✅ 核心优势

1. **极简设计** - 只需勾选和画圈,无需填写文字
2. **快速评分** - 目标5-8分钟/案例(比详细版快40%)
3. **医生友好** - 面向专业医生,去除冗余说明
4. **一页纸** - 整个评分表只有1页A4纸

### 📊 评分方式对比

| 项目 | 详细版 | 简化版 |
|------|--------|--------|
| 页数 | 10页 | 1页 |
| 评分时间 | 10-12分钟 | 5-8分钟 |
| 填写文字 | 需要 | 不需要 |
| 适用人员 | 所有评测员 | 专业医生 |
| 质控复核 | 详细 | 简化 |

## 🎯 5分钟快速评分流程

### Step 1: 准备 (30秒)

1. 拿到案例编号(如 #037)
2. 填写评测员姓名和日期
3. 准备原始问诊记录对照

### Step 2: 信息完整性 (1分钟)

1. 快速浏览模型输出
2. 勾选包含的10个关键信息
3. 数缺失几个,在对应分数上画圈

**示例**:
```
全部都有 → 画圈20分
缺1个 → 画圈18分
缺2个 → 画圈16分
```

### Step 3: 信息准确性 (1.5分钟)

1. 对照原始记录,逐项核对
2. 发现错误就勾选对应类型
3. 计算扣分: 严重×5 + 中等×3 + 轻微×1

**示例**:
```
发现1个严重错误(病程10年,实际12年)
→ 勾选"病程时间错误"
→ 1个×5 = 扣5分
→ 得分: 25-5=20分
```

### Step 4: 结构与组织 (30秒)

1. 整体看一眼结构
2. 选择最接近的档次(A/B/C/D/E)
3. 在对应评级上画圈

**判断标准**:
- A(15分): 分节清晰,格式规范,一眼找到信息
- B(12分): 基本清晰,小问题
- C(9分): 分节有但不够清晰
- D(6分): 混乱
- E(3分): 无结构

### Step 5: 临床相关性 (1分钟)

1. 检查4个项目,有就勾选
2. 数勾选了几个
3. 在对应分数上画圈

**4个检查项**:
- □ 突出重点问题
- □ 标注风险点
- □ 有"重点关注"总结
- □ 信息聚合好

**示例**:
```
勾选了4个 → 20分
勾选了3个 → 16分
勾选了2个 → 12分
```

### Step 6: 语言表达 (30秒)

1. 快速判断语言是否专业
2. 选择档次(A/B/C/D/E)
3. 画圈

**判断标准**:
- A(10分): 专业简洁,无口语
- B(8分): 基本专业
- C(6分): 有口语化
- D(4分): 多处口语化
- E(2分): 非常不专业

### Step 7: 整体可用性 (30秒)

1. 问自己: 这份输出能直接用于临床吗?
2. 选择对应档次
3. 画圈

**5个档次**:
- 直接可用(10分): 不用改
- 稍作修改(8分): 小改
- 需要修改(6分): 一定工作量
- 大量修改(4分): 很多工作
- 不可用(2分): 重写

### Step 8: 汇总 (30秒)

1. 把6个维度的分数填入汇总表
2. 加总得到总分
3. 根据总分勾选总体评级(优秀/良好/中等/较差/很差)
4. 签名,记录耗时

## 💡 评分技巧

### 1. 信息完整性快速检查法

**10个关键点记忆口诀**:
```
基本+主诉+病程起,
四药+血糖+体重降,
住院+既往+家族史
```

**对照清单**:
1. 基本信息(4项)
2. 主诉
3. 病程
4. 起病经过
5. 4种药+频次
6. 血糖
7. 体重变化
8. 住院史
9. 既往史
10. 家族史

### 2. 准确性快速检查法

**重点核对**:
- 数值: 年龄、身高、体重、血糖、病程时间
- 药物: 4种药名是否正确
- 时间: 12年、5个月前、2个月

**常见错误**:
- 病程写成10年(实际12年) → 严重错误
- 血糖无单位 → 轻微错误
- 时间"3个月前"(实际5个月) → 中等错误

### 3. 档次选择技巧

**当不确定在两个档次之间时**:
- 偏向中间档(如A和C之间选B)
- 第一印象很重要
- 参考示例案例

### 4. 时间控制

**每个维度时间分配**:
```
完整性: 1分钟
准确性: 1.5分钟
结构: 30秒
临床性: 1分钟
语言: 30秒
可用性: 30秒
汇总: 30秒
-----
总计: 6分钟
```

**如果超时**:
- 前几个案例慢一点正常(培养手感)
- 熟练后应控制在5-8分钟
- 超过10分钟说明过于仔细,适当放松

## ⚠️ 常见错误

### 错误1: 过于纠结细节

**问题**: 花5分钟纠结该给18分还是20分
**解决**: 第一印象选择,不要反复修改

### 错误2: 标准不一致

**问题**: 第1个案例严格,第70个案例宽松
**解决**:
- 每评10个案例,休息并回顾标准
- 参考示例案例校准

### 错误3: 不敢打极端分数

**问题**: 所有案例都在70-85分
**解决**:
- 真正优秀的给90+分
- 真正很差的给<60分
- 参考: 健康分布是正态分布

### 错误4: 忘记对照原始记录

**问题**: 仅看模型输出就打分
**解决**:
- 准确性评分必须对照原始记录
- 完整性评分也需要对照

## 📝 评分表填写注意事项

### 1. 使用工具

**推荐**:
- 黑色签字笔(便于扫描)
- 铅笔(可修改,但最后描黑)

**不推荐**:
- 红笔(扫描不清)
- 圆珠笔(容易模糊)

### 2. 标记方式

**清晰的标记**:
```
✓ - 勾选
✗ - 未勾选/错误
○ 涂黑 - 选中的分数
```

**不清晰的标记**:
```
√ 潦草的勾
◯ 半涂的圈
? 模糊的符号
```

### 3. 修改方式

**正确修改**:
```
原来: ○ 涂黑
改为: 用双线划掉,在新位置涂黑
```

**错误修改**:
```
用涂改液 (扫描不清)
擦橡皮擦到破 (纸张破损)
```

## 📊 批量评分组织

### 每天评分计划

**每天20个案例** (约2-3小时):
```
上午: 10个案例 (1.5小时)
  - 休息15分钟
下午: 10个案例 (1.5小时)
```

**每批次建议**:
- 连续评5个案例
- 休息5-10分钟
- 再评5个

### 每周进度追踪

| 天数 | 案例数 | 累计 | 进度 |
|------|--------|------|------|
| 周一 | 15 | 15 | 21% |
| 周二 | 20 | 35 | 50% |
| 周三 | 20 | 55 | 79% |
| 周四 | 15 | 70 | 100% |

**目标**: 4天内完成70个案例

## 🔍 质量自检

### 每10个案例后自检

**检查点**:
- [ ] 评分标准前后一致吗?
- [ ] 有无疲劳导致放松标准?
- [ ] 分数分布合理吗?(不全都70-80分)
- [ ] 所有项目都填写了吗?

**分数分布参考**:
```
90-100分: 1-2个 (10-20%)
80-89分: 3-4个 (30-40%)
70-79分: 3-4个 (30-40%)
60-69分: 1-2个 (10-20%)
<60分: 0-1个 (0-10%)
```

### 重测一致性检验

**方法**:
- 随机抽取3个已评案例
- 隔2-3天重新评测
- 对比两次评分

**一致性标准**:
- 差异<5分: 优秀
- 差异5-10分: 可接受
- 差异>10分: 需要调整标准

## 📞 遇到问题怎么办

### 情况1: 模型输出看不懂

**处理**:
- 标记为"需复核"
- 暂打低分(40-50分)
- 在备注栏写"输出混乱"
- 提交给质控员

### 情况2: 不确定打多少分

**处理**:
- 选择中间档次
- 在备注栏写"边界案例"
- 继续下一个
- 质控时讨论

### 情况3: 原始记录有歧义

**处理**:
- 按照常见理解评分
- 在备注栏注明
- 不要花太多时间纠结

### 情况4: 前后评分标准不一致

**处理**:
- 立即暂停
- 回顾前面5-10个案例
- 必要时调整之前的评分
- 休息后继续

## 📈 效率提升建议

### 熟练后可达到的速度

- 第1周: 8-10分钟/案例
- 第2周: 6-8分钟/案例
- 第3周: 5-7分钟/案例

### 提速技巧

1. **使用清单**: 打印10个关键点清单贴在桌上
2. **快速标记**: 边看边勾,不要看完再勾
3. **第一印象**: 档次选择不要反复改
4. **批量操作**: 5个案例一批,节奏感更好

---

**使用建议**:
- 第一次评分前,先用示例案例练习2-3遍
- 正式评分的前3个案例可以慢一点
- 从第4个开始控制时间
- 每天结束时回顾当天评分,保持一致性

**目标**: 熟练评测员应在5-8分钟内完成一个案例,保持高质量和高效率的平衡。
