# LLM医疗病历信息提取 - 人工评测详细指南

## 一、评测目标

通过医疗专业人员对7个LLM模型生成的病历信息进行全面、客观的人工评测,从**临床实用性**角度评估模型表现。

## 二、评测团队组建

### 2.1 人员要求

**主评测员 (3-5人)**
- **必需资质**:
  - 临床医生(主治医师及以上职称)
  - 5年以上临床经验
  - 熟悉糖尿病等慢性病诊疗流程

- **优先条件**:
  - 有问诊记录整理经验
  - 熟悉电子病历系统
  - 对AI应用有一定了解

**质量控制员 (1人)**
- 高年资医生或医疗信息专家
- 负责一致性检验和争议裁决

**数据管理员 (1人)**
- 负责数据整理、统计分析
- 可为技术人员或研究助理

### 2.2 团队培训

**培训时长**: 2-3小时

**培训内容**:
1. 项目背景介绍(30分钟)
   - LLM在医疗领域的应用
   - 本项目的目标和意义
   - 评测结果的用途

2. 评测标准讲解(60分钟)
   - 详细讲解每个评分维度
   - 提供打分示例和反例
   - 讨论边界案例

3. 评测流程演示(30分钟)
   - 演示评测界面使用
   - 模拟评测1-2个案例
   - 答疑解惑

4. 试评测(30分钟)
   - 每人独立评测3个案例
   - 对比评分差异
   - 统一标准尺度

## 三、评测维度与评分标准

### 维度1: 信息完整性 (20分)

**定义**: 模型输出是否包含所有关键医疗信息,无重要遗漏

**评分标准**:

**20分 (优秀)**:
- 所有关键信息完整提取
- 包括: 主诉、现病史(起病、进展、治疗)、既往史、家族史、用药史、检查结果
- 次要信息(如生活方式)也有涉及
- 无重要遗漏

**15-19分 (良好)**:
- 关键信息基本完整
- 遗漏1-2个次要信息点
- 如: 缺少烟酒史、缺少具体用药剂量等

**10-14分 (中等)**:
- 关键信息大部分完整
- 遗漏3-4个信息点,或1个较重要信息
- 如: 缺少家族史、缺少部分检查结果

**5-9分 (较差)**:
- 关键信息不完整
- 遗漏多个重要信息或多处次要信息
- 如: 病程描述不清、用药信息缺失

**0-4分 (很差)**:
- 大量关键信息缺失
- 无法形成完整的病史认识

**评分要点**:
- ✓ 对照原始问诊记录,逐项核对
- ✓ 重要信息(诊断相关)权重更高
- ✓ 次要信息可适当宽容

**示例**:

```
原始问诊: "患者12年前发现糖尿病,曾用胰岛素,现用二甲双胍早晚各1次、
司美格鲁肽每周1次、格列齐特早2粒、多格列艾汀早晚各1次,空腹血糖11,
5个月前住院,出院后2月内体重降10余斤"

模型A输出 (18分):
✓ 病程12年
✓ 既往用胰岛素
✓ 当前4种药物+剂量频次
✓ 空腹血糖11
✓ 5月前住院
✓ 体重下降
✗ 未提及餐后血糖未监测

模型B输出 (12分):
✓ 病程12年
✓ 当前用药(仅提及药名,无剂量)
✓ 血糖升高(未写数值)
✗ 未提住院史
✗ 未提体重变化
✗ 未提既往胰岛素
```

### 维度2: 信息准确性 (25分)

**定义**: 模型输出的信息与原始问诊记录是否一致,无错误或曲解

**评分标准**:

**25分 (优秀)**:
- 所有信息准确无误
- 数值、时间、药名、剂量完全正确
- 无任何曲解或误读

**20-24分 (良好)**:
- 绝大部分信息准确
- 有1-2处轻微错误或不精确表达
- 如: "50多岁"误写为"50岁","早晚各1次"误写为"每日2次"

**15-19分 (中等)**:
- 大部分信息准确
- 有3-4处错误,或1处较严重错误
- 如: 药物剂量错误、时间节点混淆

**10-14分 (较差)**:
- 准确性不足
- 多处错误,或数处严重错误
- 如: 病程时间错误、药物名称错误、血糖数值错误

**0-9分 (很差)**:
- 大量错误
- 关键信息严重失真

**常见错误类型**:

| 错误类型 | 严重程度 | 扣分参考 |
|----------|----------|----------|
| 数值完全错误 | 严重 | -5分 |
| 数值不精确(范围vs精确值) | 轻微 | -1分 |
| 药名错误 | 严重 | -5分 |
| 剂量/频次错误 | 中等 | -3分 |
| 时间点错误(1-2年) | 中等 | -3分 |
| 时间点错误(>5年) | 严重 | -5分 |
| 因果关系颠倒 | 严重 | -4分 |
| 无中生有 | 严重 | -5分 |

**示例**:

```
原始: "空腹血糖11mmol/L,未监测餐后血糖"

模型A (扣1分,24分):
"空腹血糖约11mmol/L" ✓ (不精确但可接受)

模型B (扣3分,22分):
"空腹血糖11mg/dL" ✗ (单位错误,中等严重)

模型C (扣5分,20分):
"空腹血糖7mmol/L" ✗ (数值完全错误,严重)

模型D (扣5分,20分):
"空腹血糖11,餐后血糖14" ✗ (无中生有,严重)
```

### 维度3: 结构与组织 (15分)

**定义**: 信息组织是否清晰、逻辑,方便临床使用

**评分标准**:

**15分 (优秀)**:
- 分节清晰(基本信息、主诉、现病史、既往史等)
- 层次分明,使用适当的标题、列表
- 时间线清晰,逻辑流畅
- 一目了然,快速定位信息

**12-14分 (良好)**:
- 分节基本清晰
- 个别部分组织略显混乱
- 整体可用性良好

**9-11分 (中等)**:
- 分节存在但不够清晰
- 部分信息归类不当
- 需要仔细阅读才能找到信息

**6-8分 (较差)**:
- 分节混乱或缺失
- 信息堆砌,逻辑不清
- 难以快速获取关键信息

**0-5分 (很差)**:
- 完全无结构
- 信息杂乱无章

**评分要点**:
- ✓ 站在临床医生角度,评估"好不好用"
- ✓ 重点看能否快速找到需要的信息
- ✓ 时间线清晰度很重要

**优秀案例**:
```markdown
# 患者1 - 基本信息
- 性别: 男
- 年龄: 50-55岁
- 身高: 182cm
- 体重: 64kg

## 主诉
配药

## 现病史
### 起病
- 12年前因小便泡沫多就诊
- 发现血糖升高,诊断为糖尿病

### 治疗经过
- 既往使用胰岛素(已停用)
- 当前方案:
  - 二甲双胍 早晚各1次
  - 司美格鲁肽 每周1次
  - 格列齐特 早2粒
  - 多格列艾汀 早晚各1次

### 近期变化
- 5个月前住院(血糖不稳定)
- 出院后2月内体重下降10余斤
- 近期体重稳定

## 既往史
- 高甘油三酯血症(未服药)

## 家族史
- 母亲、舅舅、哥哥均有糖尿病
```

**较差案例**:
```
患者男,50-55岁,182cm,64kg。12年前小便泡沫多,血糖高,
糖尿病。用过胰岛素,现在二甲双胍、司美格鲁肽、格列齐特、
多格列艾汀。血糖11。5个月前住院,2个月瘦了10斤。
母亲舅舅哥哥糖尿病。甘油三酯高。
```

### 维度4: 临床相关性 (20分)

**定义**: 输出是否突出临床相关的重要信息,有助于诊疗决策

**评分标准**:

**20分 (优秀)**:
- 重点信息突出(如血糖控制不佳、体重下降等警示信息)
- 相关信息聚合(如所有用药集中列出)
- 有助于快速形成诊疗思路
- 关键风险点有标注或强调

**16-19分 (良好)**:
- 重要信息基本突出
- 信息聚合较好
- 个别关键点不够明显

**12-15分 (中等)**:
- 信息平铺,重点不够突出
- 需要医生自行筛选关键信息
- 对诊疗决策帮助有限

**8-11分 (较差)**:
- 重要信息淹没在次要信息中
- 缺少临床思维
- 对诊疗决策帮助很小

**0-7分 (很差)**:
- 完全无临床导向
- 信息堆砌,无法支持决策

**临床相关性检查清单**:

| 检查项 | 是否突出 | 权重 |
|--------|----------|------|
| 血糖控制情况(当前数值+控制目标) | ✓/✗ | 高 |
| 体重变化(尤其异常下降) | ✓/✗ | 高 |
| 用药方案(完整+剂量) | ✓/✗ | 高 |
| 并发症线索(泡沫尿→肾病?) | ✓/✗ | 高 |
| 家族史(影响治疗策略) | ✓/✗ | 中 |
| 既往住院史(病情严重程度) | ✓/✗ | 中 |
| 依从性问题(用药不详→未规律?) | ✓/✗ | 中 |

**优秀案例特征**:
```markdown
## ⚠️ 重点关注

1. **血糖控制不佳**: 当前空腹血糖11 mmol/L (目标<7),4联用药仍控制不良
2. **体重异常下降**: 2个月内下降10余斤,需排除血糖控制不佳导致
3. **疑似肾脏并发症**: 小便持续泡沫(12年),建议检查尿蛋白
4. **强家族史**: 母系及同胞多人患病,预后需密切关注

## 当前用药方案
[详细列出...]
```

### 维度5: 语言表达 (10分)

**定义**: 语言是否专业、简洁、流畅,无赘述或歧义

**评分标准**:

**10分 (优秀)**:
- 医学术语使用准确
- 表达简洁明了,无冗余
- 语句通顺,无语病
- 专业性强

**8-9分 (良好)**:
- 表达基本专业流畅
- 个别用词不够精确或略显冗余

**6-7分 (中等)**:
- 表达基本清楚
- 有一定赘述或用词不当
- 专业性一般

**4-5分 (较差)**:
- 表达不够清晰
- 多处赘述或用词错误
- 阅读费力

**0-3分 (很差)**:
- 表达混乱,语病多
- 专业性差

**常见问题**:

❌ **冗余表达**:
- "患者的糖尿病疾病" → "糖尿病"
- "进行了住院治疗" → "住院治疗"

❌ **口语化**:
- "有点高" → "升高"
- "吃药" → "服药"

❌ **不专业**:
- "尿里有泡泡" → "小便泡沫多"
- "血糖不好" → "血糖控制不佳"

✓ **专业简洁**:
- "确诊2型糖尿病12年"
- "近期空腹血糖11 mmol/L"
- "口服降糖药物4种"

---

### 维度6: 整体可用性 (10分)

**定义**: 综合评估该输出在实际临床工作中的可用程度

**评分标准**:

**10分**: 可直接用于临床,无需修改
**8-9分**: 稍作修改即可使用
**6-7分**: 需要一定修改才能使用
**4-5分**: 需要大量修改
**0-3分**: 基本不可用,需要重写

**评分问题**:
1. 如果你是接诊医生,这份病历能否帮助你快速了解患者情况?
2. 是否需要再次查看原始问诊记录?
3. 能否基于此病历做出初步诊疗计划?

## 四、评测流程

### 4.1 评测前准备

**评测材料分发**:
1. 为每位评测员准备:
   - 评测案例包(70个案例,盲化处理)
   - 原始问诊记录(对照材料)
   - 评分表(纸质或电子)
   - 评分指南(本文档)

**盲化要求**:
- 所有输出随机编号(如 #001-#070)
- 不显示模型名称
- 打乱顺序(同一患者的7个模型输出不连续出现)

### 4.2 评测执行

**评测顺序**:
1. 阅读原始问诊记录(1-2分钟)
   - 熟悉患者基本情况
   - 标记关键信息点

2. 阅读模型输出(2-3分钟)
   - 完整阅读,不急于打分
   - 标注优点和问题

3. 逐维度评分(3-5分钟)
   - 按照6个维度逐一评分
   - 参考评分标准
   - 在评分表上记录分数和理由

4. 填写总评(1-2分钟)
   - 综合评价
   - 突出优点和主要问题
   - 给出改进建议

**单个案例评测时间**: 约10分钟
**单人评测70个案例**: 约12小时(可分2-3天完成)

### 4.4 质量控制

**一致性检验**:
- 每10个案例,插入1个重复案例(评测员不知情)
- 计算同一评测员对重复案例的评分一致性
- 要求重测一致性 > 0.8 (ICC)

**评测员间一致性**:
- 随机抽取10个案例,所有评测员都评测
- 计算评测员间一致性(Kappa系数)
- 要求 Kappa > 0.7

**争议案例处理**:
- 同一案例不同评测员评分差异 > 15分,列为争议案例
- 由质量控制员组织讨论
- 达成共识或取中位数

### 4.5 评测进度管理

**每周进度**:
- 每位评测员每周完成20-25个案例
- 每周五提交评分表
- 数据管理员录入系统

**总体时间轴**:
- 第1周: 培训 + 试评测
- 第2-4周: 正式评测(每周25个)
- 第5周: 争议案例处理 + 数据整理
- 第6周: 统计分析 + 报告撰写

## 五、数据分析方法

### 5.1 描述性统计

**每个模型计算**:
- 平均分(总分及各维度)
- 标准差(稳定性)
- 中位数
- 最小值/最大值
- 四分位数

### 5.2 模型对比

**统计检验**:
- 方差分析(ANOVA): 比较7个模型总分差异
- 事后检验(Post-hoc): 两两比较
- 显著性水平: p < 0.05

**效应量**:
- Cohen's d: 评估差异的实际意义
- d > 0.5 为中等差异, d > 0.8 为大差异

### 5.3 一致性分析

**评测员间一致性**:
- 组内相关系数(ICC): > 0.75 为良好
- Fleiss' Kappa: > 0.6 为实质性一致

**重测信度**:
- Pearson相关系数: > 0.8
- 配对t检验: p > 0.05(无系统性偏差)

### 5.4 案例分析

**最佳/最差案例识别**:
- 每个模型选出得分最高的3个案例
- 每个模型选出得分最低的3个案例
- 分析共性和差异

**错误类型统计**:
- 统计各类错误出现频率
- 识别每个模型的高发错误

## 六、评测报告结构

### 6.1 执行摘要

- 评测背景和目的
- 评测方法简述
- 关键发现(3-5条)
- 最佳模型推荐

### 6.2 模型排名

**综合排名表**:
| 排名 | 模型 | 总分均值 | 标准差 | 95%置信区间 |
|------|------|----------|--------|-------------|
| 1 | XXX | 85.3 | 5.2 | [83.2, 87.4] |
| ... | ... | ... | ... | ... |

**各维度排名**:
- 6个维度分别排名
- 识别每个模型的优势维度

### 6.3 详细分析

**每个模型分析** (包括):
1. 综合表现概述
2. 各维度得分及排名
3. 典型优点(带案例)
4. 典型问题(带案例)
5. 适用场景建议

## 七、评测员手册

### 7.1 评分原则

**客观性**:
- 基于标准,不受个人喜好影响
- 同等错误给予同等扣分

**一致性**:
- 前后评分标准保持一致
- 避免疲劳导致的松紧变化

**严谨性**:
- 仔细核对每一个信息点
- 不遗漏任何错误

**临床导向**:
- 站在临床医生角度评估
- 重视实用性

### 7.2 常见问题

**Q1: 模型输出格式与标准不同怎么办?**
A: 只要信息完整准确,格式可以有差异。在"结构与组织"维度评估格式优劣。

**Q2: 模型提取了额外合理推断怎么办?**
A: 如果推断合理(如根据症状推测可能的并发症),在"临床相关性"中加分。
   如果是无中生有的信息,在"准确性"中扣分。

**Q3: 两个模型得分相同时如何排名?**
A: 优先考虑标准差小的(更稳定),其次看临床相关性高的。

**Q4: 评分时间超过10分钟正常吗?**
A: 前几个案例可能较慢,熟练后应控制在10分钟内。过慢可能标准过严。

**Q5: 不确定打多少分怎么办?**
A: 参考评分标准中的描述,选择最接近的档次。记录不确定原因,供质控审查。

### 7.3 评分技巧

**信息完整性评分**:
1. 先列出原始记录的所有信息点(10-15个)
2. 逐一核对模型输出
3. 计算覆盖率: 覆盖9-10个(20分),7-8个(15-19分),5-6个(10-14分)

**信息准确性评分**:
1. 从25分开始
2. 发现一个错误,根据严重程度扣分
3. 轻微错误(-1), 中等错误(-3), 严重错误(-5)

**临床相关性评分**:
1. 检查"临床相关性清单"中的7个项目
2. 每突出1个重点+2-3分
3. 信息组织有助于决策+2-3分
