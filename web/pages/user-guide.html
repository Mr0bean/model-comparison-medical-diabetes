📋 系统概述

医疗模型评测系统用于评估不同AI模型对医疗病例的诊断质量，支持10个患者病例对8个AI模型的全面评测（共80份评测）。

访问地址：http://ruan.etodo.top

🚀 快速开始

1. 获取访问权限

访问系统需要完成码，获取方式：

  1. 访问主页 http://ruan.etodo.top
  2. 点击"申请完成码"按钮
  3. 填写您的邮箱地址
  4. 系统自动生成8位完成码（如：EVAL1234）
  5. 保存好您的完成码，用于登录和查看进度
⚠️ 重要提示：请妥善保管您的完成码，遗失后无法找回！

2. 开始评测

  1. 在主页输入您的完成码
  2. 点击"开始评测"进入评测界面
  3. 选择患者（P001-P010）
  4. 查看AI模型对话记录
  5. 点击右下角📝按钮打开评测窗口

📝 评测流程

第一步：选择患者和模型
  • 页面顶部选择患者（如P001）
  • 使用"上一个模型"/"下一个模型"按钮切换AI模型
  • 当前模型名称显示在右上角
第二步：查看对话记录

对话窗口展示：

  • 🟦 蓝色气泡：患者消息
  • 🟩 绿色气泡：AI模型回复

仔细阅读完整对话，了解AI的诊断过程和结果。

第三步：打开评测窗口

点击右下角的📝浮动按钮，打开评测窗口。

评测窗口功能：

  • 拖动：拖拽标题栏移动窗口位置
  • 调整大小：拖拽右下角调整窗口大小
  • 最小化：点击 − 按钮最小化窗口
  • 关闭：点击 × 按钮关闭窗口
  • 字体调整：使用 A⁺ A⁻ 按钮调整字体大小
第四步：评分

对5个维度进行评分，每个维度1-5星。评分标准详见下一节。

第五步：填写评语
  • 每个维度可以添加评语（可选），说明评分理由
  • 在"总体评价"栏填写综合评语（可选）
第六步：保存提交

点击"保存评测"按钮：

  • ✅ 自动保存到本地
  • ✅ 提交到服务器
  • ✅ 更新完成进度
  • ✅ 自动关闭评测窗口

📊 评分标准

对3个维度进行评分，每个维度1-5星，系统自动转换为百分制（满分100分）：

维度 评分要点 权重 满分
① 准确性
  • 是否与对话一致，没有添加或错误信息
  • 数值准确（血糖、糖化、体重等）
  • 药物准确（药名、剂量、用法）
  • 时间轴准确（发病时间、就诊时间）
  • 诊断和病程描述准确
  • 固定扣分项：不能把糖尿病写入既往史
 40% 40分
(5星×8)
② 完整性
  • 主诉：症状 + 时间
  • 现病史：诊断、治疗变化、近期情况、并发症
  • 既往史：高血压/高血脂/冠心病等
  • 家族史：糖尿病家族史
  • 每缺少一个重要模块扣2-3分
 35% 35分
(5星×7)
③ 规范性
  • 医学术语：避免口语化表达
  • 结构清晰：模块齐全、顺序正确
  • 药物规范：药名、剂量、用法写法规范
  • 时间明确：时间描述清晰准确
 25% 25分
(5星×5)

评分建议

  • ⭐ (1星)：非常差 - 有严重问题
  • ⭐⭐ (2星)：较差 - 有较多问题
  • ⭐⭐⭐ (3星)：一般 - 基本满足要求
  • ⭐⭐⭐⭐ (4星)：良好 - 表现出色
  • ⭐⭐⭐⭐⭐ (5星)：优秀 - 几乎完美
💡 总分计算：
总分 = 准确性分数 + 完整性分数 + 规范性分数
= (准确性星级×8) + (完整性星级×7) + (规范性星级×5)
满分100分

📈 查看进度

方法1：评测页面右上角

显示格式：已完成/总数 (百分比%)

示例：25/80 (31.25%)

方法2：主页查询

  1. 返回主页 http://ruan.etodo.top
  2. 输入完成码
  3. 点击"查看进度"
  4. 查看详细完成情况

⚙️ 功能说明

自动保存

评分过程中，系统每500ms自动保存到本地，防止数据丢失。

进度跟踪

  • 总计80份评测：10个患者 × 8个模型
  • 实时显示完成进度
  • 支持断点续评（刷新页面后继续）

数据持久化

  • 本地存储：使用localStorage保存评测数据
  • 服务器同步：实时提交到MongoDB数据库
  • 双重保障：即使网络中断，本地数据不丢失

💡 使用技巧

  1. 建议顺序评测：按患者编号P001→P010依次完成，避免遗漏
  2. 认真阅读对话：完整查看AI回复后再评分，确保评价准确
  3. 参考评分标准：根据权重合理分配分数
  4. 保存完成码：妥善保管完成码，用于查看进度和继续评测
  5. 定期查看进度：确认评测数据已成功提交

❓ 常见问题

Q1: 忘记完成码怎么办？

A: 完成码无法找回，建议：

  • 将完成码保存在邮箱中
  • 截图保存完成码
  • 记录在备忘录中

Q2: 评测数据会丢失吗？

A: 不会。系统采用双重保存机制：

  • 本地localStorage存储
  • 服务器MongoDB存储
  • 自动保存功能

Q3: 可以修改已提交的评测吗？

A: 可以。重新打开相同的患者-模型组合，修改后点击保存即可覆盖。

Q4: 右下角的📝按钮不见了？

A: 如果评测窗口已打开，浮动按钮会自动隐藏。关闭评测窗口后会重新显示。

Q5: 必须一次性完成所有评测吗？

A: 不需要。支持分批评测，系统会自动记录进度，下次登录可继续。
